    How To Fix HUSMM1616ASS201 Inaccessible Sectors

    Issue:

    When writing or reading to a HUSMM1616ASS201 drive, an error occurs saying “(sectors inaccessible 3028948992 to 3028951039)”.

    Solution:

    Instruct the drive that the last usable sector is before the inaccessible chunk of sectors in the drive. The inaccessible part of the drive is at the very end of the drive. When you set the last usable sector, you will not lose any usable space. After the solution is implemented, the OS will see the drive’s reported usable space as smaller. This is normal. You are making the drive not report that errored chunk, therefore it will not use it and cause errors. This guide will cover how to fix inaccessible sectors on HUSMM1616ASS201 SSDs on both Linux and Windows.

    Linux

    Warning - Data Loss

    These following commands will DESTROY any data on the drive. PLEASE backup any important data on the drive immediately.

    Run the command to view the currently connected drives: 

    `sudo sg_scan`

    For example purposes, lets say the drive is /dev/sdx – please use the correct letter for your drive.

    Warning

    This command erases all the data on the drive, please verify that this is the affected drive!

    sudo sg_format --format --size=512 /dev/sdx

    After this command, please power cycle the drive by physically disconnecting it from the backplane. After it has been disconnected for 30 seconds, please reconnect the drive. Once the drive has been reconnected and recognized, please run sudo sg_scan this is to verify the drive is still the same letter. This next command sets the last useable sector right before the errored chunk.

    sudo sg_format --resize - -count=3028948982 /dev/sdx

    please power cycle the drive once more. to verify that the command has worked, you can issue the following command

    sudo sg_readcap -a /dev/sdx

    You should now see the number of logical blocks: 3028948982 if so, you have done it correctly and your drive is ready to be used.

    Windows:

    This works for Windows 10, Windows Server 16 and 19. First navigate to sg3 utils Alternatively, google “sg3_utils” and it should be the first link, you will find it.

    Then go to “Download and Build” and scroll about half way down the page to find the Windows Zip packages.

    Download the appropriate zip for your system. Locate your download then extract it to a folder. Open a command prompt with administrator access.

    Change directory of the command prompt to the “exes” folder inside of the unzipped sg3_utils folder

    cd C:\Users\admin\Downloads\sg3_utils-1.45mgw64/exes

    From here you can run the first command to see connected drives:

    sg_scan

    you should see your drive listed as a physical disk: PD for this example, I will refer to it as PD9 – you should use the correct number for your drive. you can check the drives stats with sg_readcap PD9

    Warning - Data Loss

    These following commands will DESTROY any data on the drive, PLEASE backup any important data on the drive immediately

    run the command to view the currently connected drives: sg_scan

    For example purposes, let’s say the drive is PD9 – you should use the correct number for your drive.

    Warning

    This command erases all the data on the drive, please verify that this is the affected drive.

    sg_format - -format - -size=512 PD9

    After this command, please power cycle the drive by physically disconnecting it from the backplane. After it has been disconnected for 30 seconds, please reconnect the drive. Once the drive has been reconnected and recognized, please run sg_scan This is to verify the drive is still the same letter.

    This next command sets the last useable sector right before the errored chunk.

    sg_format - -resize - -count=3028948982 PD9

    Please power cycle the drive once more. To verify that the command has worked, you can issue the following command

    sg_readcap -a PD9

    You should now see the number of logical blocks: 3028948982 If so, you have done it correctly and your drive is ready to be used.
